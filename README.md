##⚡ Extreme Spoof Tester (Smart Edition)

<img width="634" height="933" alt="image" src="https://github.com/user-attachments/assets/e62ad642-0872-415d-9c65-85d696f1b2e6" />


این ابزار برای بررسی امنیت شبکه و سنجش آسیب‌پذیری دیتاسنترها در برابر حملات IP Spoofing (جعل آی‌پي) طراحی شده است. برنامه به صورت کاملاً هوشمند کارت‌های شبکه سیستم را اسکن کرده، نقش سرور (فرستنده/گیرنده) را تشخیص می‌دهد و راهنمایی‌های لازم را در ترمینال چاپ می‌کند


---
### ⚙️ ساختار فایل تنظیمات (config.json)


برای شروع، فایل config.json را در کنار فایل اجرایی برنامه بسازید.
⚠️ نکته: فقط کافیست آی‌پي‌های واقعی دو سرور خود را جایگزین vps_1 و vps_2 کنید. ابزار خودش تشخیص می‌دهد که در حال حاضر روی کدام سرور هستید!

آی پی هایی که میخواهید ببینید آیا اسپوف میشوند یا خیر را درون test_targets بروز رسانی کنید


```
{
  "vps_1": "192.0.2.10",
  "vps_2": "198.51.100.20",
  "test_count": 5,
  "test_targets": [
    "192.0.2.0/24",
    "203.0.113.5"
  ]
}
```

---

فایل ساخته شده را دانلود کنید یا آنرا کامپایل کنید

🚀 راهنمای قدم به قدم اجرای تستبرنامه باید روی هر دو سرور کامپایل و مستقر شود. به دلیل استفاده از سۆکت‌های لایه ۳ (Raw Sockets) برای جعل هدرهای شبکه، اجرای برنامه حتماً نیاز به دسترسی ریشه (root/sudo) دارد.1.کامپایل پروژه:روی هر دو سرور.پروژه را با دستور زیر به صورت بهینه کامپایل کنید:

```
cargo build --release

```
----

### 2.اجرای سرور گیرنده (Receiver):روی سرور اول - Mode R.ابتدا وارد سرور اول (مثلاً vps_1) شده و ابزار را در حالت گیرنده روشن کنید تا آماده شنود پکت‌ها شود:

```
sudo ./extreme_spoof_test r
```

برنامه به طور خودکار تشخیص می‌دهد که این سیستم گیرنده است و منتظر فرستنده می‌ماند.


---

### برنامه به طور خودکار تشخیص می‌دهد که این سیستم گیرنده است و منتظر فرستنده می‌ماند.3.اجرای سرور فرستنده (Sender):روی سرور دوم - Mode S.حالا وارد سرور دوم (vps_2) شده و دستور ارسال پکت‌های جعلی را صادر کنید:

```
sudo ./extreme_spoof_test s
```
----

### 📊 تحلیل نتایج (Logs & Output)

پس از اتمام کار فرستنده، خروجی سرور گیرنده (r) را بررسی کنید:

اگر عبارات زیر را مشاهده کردید، یعنی دیتاسنتر یا ارائه‌دهنده شبکه شما پکت‌های با سورس جعلی را فیلتر نمی‌کند و شبکه آسیب‌پذیر (Vulnerable) است:

```
[12:05:22] 🔥 [VULNERABLE] Spoof Allowed From IP: 192.0.2.1
[12:05:22] 🔥 [VULNERABLE] Spoof Allowed From IP: 192.0.2.2
```

همزمان تمام آی‌پي‌های موفق در یک فایل متنی مجزا (مانند working_ips_20260524.txt) جهت خروجی گرفتن و مستندسازی ذخیره می‌شوند.
